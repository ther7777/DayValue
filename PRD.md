# DayValue (日均成本摊销计算器) - 产品需求文档

## 📌 版本记录 (Changelog)
* **v1.0 (当前开发中)**：引入“赎身制”双轨看板、跑通核心公式、基础 CRUD、像素风 UI 框架打底。
* **v2.0 (实现中)**：订阅周期新增“季付”、支持自定义分类/图标（分类管理）、新增统计页（2 个饼图）、赎身成功 1.5s 动画、种子数据统一标记“(示例)”。

## 1. 产品定位与视觉风格
* **核心价值观**：不仅是计算摊销，更是一个“消费观念矫正器”。鼓励全款买断，惩罚分期消费。
* **核心机制（赎身制）**：分期未还完的物品处于"未赎身"的吸血状态，不提供摊销降价的正向情绪价值；只有全款或还清欠款，物品才算真正属于用户，进入"买断资产"的正向循环。
* **UI/UX 风格设定**：清新的像素风 (Fresh Pixel Art)。整体色彩明快，交互轻量。

## 2. 核心实体与计算逻辑 (双轨制)
*(注：v1.0 数据库建表时必须包含 icon, category, is_installment 等新增字段)*

### 2.1 轨迹一：一次性资产 (One-Time Items) - 包含“赎身”状态机
* **基础字段**：名称、分类、图标、总金额、购买日期、残值。
* **分期专属字段**：`is_installment` (是否分期/先用后付)、`installment_months` (分期月数)、`monthly_payment` (月供)。
* **状态机 (Status)**：
  * **状态 A【未赎身/吸血中】(分期中)**
    * **逻辑**：隐藏物理资产的“日均摊销成本”，剥夺正向反馈。
    * **计算**：强制展示为【日供】 = `monthly_payment / 30`。
      * 列表卡片（2字视图）：显示为 `日供`，与"月供"形成对仗，一眼看懂"月供打碎到每天"。
      * 详情页（展开视图）：显示为 `影子日供`，强调这是"尚未真正属于你的资产每天的影子成本"。
  * **状态 B【已赎身/在用】(全款买入 或 分期结束)**
    * **逻辑**：物品真正属于你。解锁正向反馈公式。
    * **计算**：【日均成本】 = `(总金额 - 残值) / 使用天数`。
    * *(注：使用天数自购买日开始算，但在“赎身”前不向用户展示这个计算结果)*。
  * **状态 C【已归档】(吃灰/闲置/出二手)**
    * **逻辑**：停止计日，成本锁定。

### 2.2 轨迹二：周期订阅资产 (Subscriptions) - 纯消耗
* **字段**：名称、分类、图标、周期金额、计费周期、开始日期、状态 (生效/取消)。
* **计算公式**：
  * **每日固定消耗** = 周期金额 / 30（月付） | 周期金额 / 90（季付） | 周期金额 / 365（年付）

## 4. v2.0 迭代功能（实现中）

### 4.1 种子数据示例标记
* 所有默认插入的示例物品/订阅的 `name` 统一追加 ` (示例)`，避免用户误认为真实数据。

### 4.2 订阅周期扩展
* `billing_cycle` 支持：`monthly` / `quarterly` / `yearly`。
* 日均消耗计算：月/季/年分别除以 30/90/365。

### 4.3 自定义分类与图标
* 新增 `Categories` 表：`(id, name, icon, type)`，`type` 用于区分物品/订阅分类。
* 首页入口：⚙️ 分类管理页，可新增分类（名称 + 预设图标）。
* 添加/编辑页：分类选择从数据库读取，不再写死数组。

### 4.4 数据统计图表
* 新增统计页（📊 入口），展示 2 个饼图：
  * 买断资产：分类价值占比（按 `total_price` 汇总）。
  * 每日消耗：分类金额占比（分期日供 + 订阅日均成本）。

### 4.5 “赎身成功”视觉动画
* 物品详情页点击【赎身/结清】后先确认，再弹出 Modal 播放约 1.5 秒的 Animated 文字动画。
* 动画结束后才更新数据库状态，并返回上一页。

## 3. 核心页面结构 (MVP)
* **首页 (Dashboard) - 双轨看板**：
  * **顶部标签页 1【买断资产】(绿/紫色系)**：只展示【已赎身/active】的物品。
    * Hero 区显示"今日日均成本 ↓"，配向下趋势符号，强调数字越低越好。
    * 数字已是摊销后的沉没成本，越低代表越"回本"，给用户"越用越划算"的爽感。
  * **顶部标签页 2【每日消耗】(橙色系)**：集中展示【未赎身/unredeemed】的分期物品和【周期订阅】服务。
    * Hero 区显示"今日固定流失"，使用橙色配色，强调现金流失的警醒感（非"债务"，而是"持续消耗"）。
    * 注：两条轨道的数值含义相反，故 Header 跟随 Tab 切换，**禁止在同一 Header 内并排展示两个轨道的数字**。
* **哲学问号弹窗**：Hero 区右侧放置 `(?)` 图标，点击弹出 Modal，分两段说明计算逻辑：
  * 🟩 【买断资产】：总价已锁定，用的天数越多，日均成本越低。公式：`(总价 - 残值) ÷ 使用天数`。
  * 🟥 【每日消耗】：分期未还完或持续订阅，每天一睁眼即被扣的底线开销。公式：`月供或月租 ÷ 30`。
* **分期物品赎身进度**：状态为 `unredeemed` 的卡片，额外显示像素风进度条 + "已还 X / Y 期"文字，让用户看到"终点"。
* **添加/编辑页**：
  * 表单录入。选择"一次性购买"时，必须增加一个 Switch 开关【是否分期/先用后付】。如果打开，则要求输入月供和期数。
* **详情与状态变更页**：
  * 对于分期物品，提供一键【提前结清/赎身】按钮，点击后状态变为"已赎身"，物品跃迁至买断资产列表。
  * 提供“停止使用/归档”并录入残值的操作。
