# DayValue - 产品需求文档（PRD）

> 最后更新：2026-03-05  
> 本 PRD 以 `main` 分支当前实现为准（Expo React Native + SQLite）。

## 0. 一句话
DayValue 用“赎身制”把消费分成三条轨道：**买断资产（越用越回本）** / **每日消耗（每天固定流失）** / **沉睡卡包（被商家占用的本金）**，用最少输入给用户最直观的心理反馈与行动指引。

## 1. 产品定位与视觉风格
- **核心价值观**：不仅是计算摊销，更是一个“消费观念矫正器”。鼓励全款买断、警惕分期、清理订阅、尽快消耗储值/次卡。
- **UI/UX 风格**：清新像素风（Fresh Pixel Art），色彩明快，交互轻量；重要数字在 Hero 区“先被看见”。
- **展示原则**：首页 Header 跟随 Tab 切换，**禁止在同一 Header 内并排展示不同轨道的数字**（避免语义混淆）。

## 2. 术语与状态（面向用户）
- **买断资产**：一次性资产已赎身/在用（`status = active`）的部分。
- **每日消耗**：未赎身分期（`status = unredeemed`）+ 生效订阅（`status = active`）。
- **沉睡卡包**：储值卡 / 计次卡（StoredCards）。
- **隐藏**：统一的“归档/收起”口径（数据库状态仍为 `archived`）。
  - 列表默认只展示在用数据；用户可“显示/收起已隐藏”。
  - 编辑页支持“隐藏 / 取消隐藏（恢复显示）”。

## 3. 核心实体与计算逻辑

### 3.1 一次性资产（One-Time Items）
**字段（核心）**：名称、分类、图标、总金额、购买日期、残值、状态。  
**分期字段**：是否分期、分期月数、月供、首付。

**状态机（Status）**
- **A 未赎身（unredeemed）**：分期中。隐藏日均摊销的正反馈，强制展示“影子日供”。
- **B 在用（active）**：已赎身/全款/分期结束，展示“日均成本”。
- **C 已隐藏（archived）**：停止计日，成本锁定（写入 `end_date` 与 `salvage_value`）。

**核心公式**
- 使用天数：`(endDate || today) - buyDate + 1`（最小为 1）
- 日均成本：`(total_price - salvage_value) / days_used`
- 影子日供（隐形日供）：`monthly_payment / 30`

### 3.2 周期订阅（Subscriptions）
**字段**：名称、分类、图标、周期金额、计费周期、开始日期、状态（生效/已退订）。  
**日均成本**
- 月付：`cycle_price / 30`
- 季付：`cycle_price / 90`
- 年付：`cycle_price / 365`

### 3.3 沉睡卡包（StoredCards）
**卡片类型**
- **储值卡**（金额型）：维护“总面值 / 当前剩余额度”，支持“本次扣减 / 设定剩余”。
- **计次卡**（次数型）：维护“总次数 / 当前剩余次数”，支持“打卡 -1”。

**核心概念：实际沉睡本金**
- 公式：`实际沉睡本金 = 当前剩余 × (实际支付 / 总面值)`
- 直觉：你提前把真钱交给商家，剩余未消费的部分相当于被商家占用的无息本金。

### 3.4 自定义分类（Categories）
`Categories(id, name, icon, type)`  
`type` 区分：`item` / `subscription` / `stored_card`。分类管理支持“名称 + 预设图标”。

## 4. 核心页面结构与交互

### 4.1 首页（Dashboard）
顶部三 Tab：**买断资产 / 每日消耗 / 沉睡卡包**。

**买断资产**
- Hero：`今日日均成本 ↓` + 金额（`/天`）。
- 列表：只展示 `active`；可“显示/收起已隐藏”。

**每日消耗**
- Hero：`今日固定流失` + 金额（`/天`）。
- 副标题（金额拆解）：`隐形日供 ¥X · 订阅日均 ¥Y`，让用户一秒理解总额构成。
- 列表：分期物品（未赎身）+ 生效订阅。

**沉睡卡包**
- Hero：`实际沉睡本金 ↑` + 金额。
- 列表：在用卡包；可“显示/收起已隐藏”。
- 新增：选择“储值卡 / 计次卡”。

### 4.2 分类管理（Categories）
- 三类切换：物品 / 订阅 / 卡包。
- 新增/编辑弹窗：图标网格可滚动，支持点击遮罩关闭，避免“弹窗内容过长卡死”。

### 4.3 统计页（Statistics）
三张饼图：
- 买断资产：分类价值占比（按 `total_price` 汇总）。
- 每日消耗：分类金额占比（隐形日供 + 订阅日均）。
- 沉睡卡包：分类“实际沉睡本金”占比。

**图例展示规则（有效位）**
- 内置 legend 不使用，改为自绘图例：同时展示“占比（有效位）+ 金额（两位小数）”。
- 每日消耗额外标注单位：`/天`。

### 4.4 详情页与状态变更
- 分期物品支持“赎身/结清”（弹窗确认 + 约 1.5s 动画）。
- 一次性资产支持“停止使用/隐藏”并录入残值（锁定成本）。
- 卡包支持“隐藏 / 取消隐藏（恢复显示）”。

## 5. 展示与格式化规则
- 金额统一使用 `formatCurrency`（两位小数；极小值显示 `< ¥0.01`）。
- 统计占比展示保留有效位：例如 `<0.1%`、`3.2%`、`12%`。

## 6. 关键校验规则（防“逻辑矛盾数据”）
- 一次性资产：`残值 ≤ 总金额`；残值不可为负。
- 分期资产：月供、期数必须为正；`首付 < 总金额`。
- 储值卡（金额型）：`总面值 ≥ 实际支付`；`当前余额 ≤ 总面值`；扣减金额需 `> 0`。
- 计次卡：`剩余次数 ≤ 总次数`。

## 7. 种子数据与示例标记
- 默认示例数据 `name` 统一追加 ` (示例)`，避免用户误认为真实数据。
